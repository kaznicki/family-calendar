Create `src/lib/useDragSelect.ts`:

```typescript
import { useState, useCallback } from 'react'
import { eachDayOfInterval, parseISO, format, isBefore } from 'date-fns'

export interface DragState {
  startDate: string
  endDate: string
  active: boolean
}

const INITIAL_STATE: DragState = { startDate: '', endDate: '', active: false }

interface DragSelectHandlers {
  dragState: DragState
  handlePointerDown: (e: React.PointerEvent<HTMLElement>, date: string) => void
  handlePointerMove: (e: React.PointerEvent<HTMLElement>, date: string) => void
  handlePointerUp: (e: React.PointerEvent<HTMLElement>) => {
    type: 'click' | 'drag' | 'none'
    startDate: string
    endDate: string
  }
  selectionDates: string[]
}
```

Implementation rules:
- `handlePointerDown`: call `e.currentTarget.setPointerCapture(e.pointerId)`, set `dragState = { startDate: date, endDate: date, active: true }`, record pointerDown X/Y for threshold
- `handlePointerMove`: if not active, return; update `endDate` to current date param; add `user-select: none` to body (via `document.body.style.userSelect = 'none'`)
- `handlePointerUp`: call `e.currentTarget.releasePointerCapture(e.pointerId)`, restore `document.body.style.userSelect = ''`; compute total pointer movement; if < 4px OR startDate === endDate → return `{ type: 'click', startDate, endDate }` (caller opens single-day popover); else → return `{ type: 'drag', startDate: min, endDate: max }` (caller opens multi-day popover); reset dragState to INITIAL_STATE
- `selectionDates`: computed from sorted min/max of dragState.startDate/endDate when active, else []
- Normalize date order: if endDate is before startDate (backward drag), swap in selectionDates and in the returned multi-day range

NOTE: No unit tests for useDragSelect (pointer events require real DOM; visual behavior verified in Plan 09 checkpoint). TypeScript must compile without errors.

**WeekRow.tsx multi-day rendering:**

The week row layout needs two layers:
1. Row 0 (slot 0): a 7-column CSS grid row for multi-day events. Multi-day events span across columns using `gridColumn`.
2. Rows 1-4 (slots 1-4): DayColumns with single-day events (existing structure).

Approach: use a container div with `position: relative` for the week row. The multi-day row is a `display: grid; grid-template-columns: repeat(7, 1fr)` div of fixed height h-7 at the top.

For each multi-day event in the current week (slotMap value === 0):
- Compute `segmentStart = max(parseISO(event.date), weekStart)`
- Compute `segmentEnd = min(parseISO(event.endDate!), weekEnd)`
- `colStart = getDay(segmentStart) + 1`  (getDay: 0=Sun..6=Sat)
- `colSpan = differenceInCalendarDays(segmentEnd, segmentStart) + 1`
- Render `<div style={{ gridColumn: \`\${colStart} / span \${colSpan}\` }}>` containing an EventCard

For DayColumn slot positions: the existing 5 slots now become 4 (slots 1-4). Slot 0 is handled by WeekRow's multi-day row. DayColumn should render 4 EventSlot items (indices 1-4).

For overflow: count events where slotMap.get(id) === -1 for this date. If count > 0, render a small `<div className="text-[9px] text-gray-400 text-center">+{N}</div>` after the 4 slots.
