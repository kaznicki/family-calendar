```typescript
const DAY_LABELS = ['Sun', 'Mon', 'Tue', 'Wed', 'Thu', 'Fri', 'Sat']
```

Component structure:
- Read `const snapshot = useRecurringMap()` for current entries (useSyncExternalStore, stable)
- Local state: `popover: { personId: string; dayOfWeek: number; anchorEl: HTMLElement | null; existing?: string } | null`
- Layout: fixed-position div at bottom of viewport, white bg, border-t, z-20, overflow-auto (for very small screens)
  - `className="fixed bottom-0 left-0 right-0 bg-white border-t border-gray-200 z-20"`
- Inner: compact table-like grid
  - Header row: empty label cell + 7 day labels ('Sun'..'Sat') in text-[10px] text-gray-400
  - One data row per person in PEOPLE:
    - `data-testid="person-row"`
    - Row has `bg-{person.colorToken}` class for the label cell only (not entire row — tint just the label)
    - Label cell: person.label in text-[10px] font-medium, constrained width (w-10 or w-12)
    - 7 day cells: `data-testid="recurring-cell"`, h-6, text-[9px], cursor-pointer, border-l border-gray-100
      - If `snapshot["{person.id}-{dayIndex}"]` exists: show the label text, truncated
      - If empty: show nothing (or faint '+' on hover via group-hover)
      - onClick: set popover state to { personId, dayOfWeek: dayIndex, anchorEl: e.currentTarget, existing: text | undefined }

- EventPopover rendered when popover state is set:
  - `anchorEl={popover.anchorEl}`
  - `isOpen={!!popover}`
  - `initialValues={popover.existing ? { title: popover.existing, personId: popover.personId } : undefined}`
  - `onSave={({ title }) => { ydoc.transact(() => { recurringMap.set(\`${popover.personId}-${popover.dayOfWeek}\`, title) }); setPopover(null) }}`
  - `onDelete={() => { ydoc.transact(() => { recurringMap.delete(\`${popover.personId}-${popover.dayOfWeek}\`) }); setPopover(null) }}`
  - `onClose={() => setPopover(null)}`
  - Note: personId and time are not relevant for recurring entries — the person is fixed by row; time is optional. The popover will show color chips but saves should ignore personId (the row's person is already known).

IMPORTANT: The RecurringFooter uses `position: fixed` at the bottom. CalendarGrid must account for this height — add `pb-24` (or similar) to the scrollable body div so the last calendar week is not hidden behind the footer. Alternatively, measure the footer height with a ref and apply it as padding. For simplicity, use a fixed `pb-28` on the scrollable body.
