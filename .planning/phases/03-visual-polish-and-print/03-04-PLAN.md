  <action>
    Create src/components/PrintGrid.tsx as a new file.

    PrintGrid is a simplified, non-interactive calendar rendering the next 10 weeks. No dragging, no EventPopover, no recurring footer, no sticky headers.

    Props:
    ```typescript
    interface PrintGridProps {
      weeks: WeekData[]
      eventsMap: Record<string, unknown[]>
      holidays: Record<string, boolean>
      birthdays: BirthdayEntry[]
    }
    ```

    Implementation:
    1. Import: format from date-fns; isDayWeekend, isDayHoliday, isDayBirthday, BirthdayEntry, WeekData from ../lib/dates; CalendarEvent from ../lib/people.

    2. For each week in weeks, render a block-level wrapper div with className "print-week-row" containing:
       a. A week-range label div (same format as WeekRow — use week.label)
       b. A 7-column grid div with className "print-week-grid" containing 7 DayCell divs (one per day)

    3. Each DayCell:
       - Computes isoDate via format(date, 'yyyy-MM-dd')
       - Derives bgColor with the same priority as DayColumn:
         ```typescript
         const birthday = isDayBirthday(date, birthdays)
         const holiday = isDayHoliday(isoDate, holidays)
         const weekend = isDayWeekend(date)
         const bgColor = birthday
           ? 'var(--color-birthday-bg)'
           : (holiday || weekend)
           ? 'var(--color-weekend-bg)'
           : undefined
         ```
       - Applies bgColor via inline style (NOT Tailwind dynamic class)
       - Shows date number (format(date, 'd')) in a small header
       - Shows birthday name if birthday exists: small label "Name's bday"
       - Lists events for that date from eventsMap[isoDate] — each event shown as a small colored div:
         ```tsx
         const dayEvents = (eventsMap[isoDate] ?? []) as unknown as CalendarEvent[]
         {dayEvents.slice(0, 4).map(ev => (
           <div
             key={ev.id}
             className="text-[8px] leading-tight px-0.5 truncate rounded-sm mb-0.5"
             style={{ backgroundColor: `var(--color-${ev.personId})` }}
           >
             {ev.title}
           </div>
         ))}
         ```
       - Do NOT render multi-day spanning rows — treat all events as single-day for print simplicity (PRNT-01 requires "clean grid", no specific multi-day spanning requirement for print)

    4. DayCells use these minimal classes: `border-r border-b border-gray-200 p-0.5 text-[9px]`

    5. The outer wrapper div for the whole PrintGrid: `className="text-black bg-white"` — explicit black/white for print.

    Then update src/index.css — append the following after the existing `* { box-sizing: border-box; }` block:

    ```css
    /* ─── Print utilities ─────────────────────────────────────────────────────── */

    @media screen {
      .print-only { display: none !important; }
    }

    @media print {
      .screen-only { display: none !important; }
      .print-only { display: block !important; }

      /* 7-column print grid — use fixed % not fr (fr collapses in print engines) */
      .print-week-grid {
        display: grid;
        grid-template-columns: repeat(7, 14%);
        border-left: 1px solid #e5e7eb;
        border-top: 1px solid #e5e7eb;
      }

      /* Prevent page breaks mid-week */
      .print-week-row {
        break-inside: avoid;
        page-break-inside: avoid;
      }

      /* Week label styling */
      .print-week-label {
        font-size: 9px;
        color: #6b7280;
        padding: 2px 4px;
      }

      /* Landscape and margins */
      @page {
        size: landscape;
        margin: 10mm 8mm;
      }
    }
    ```

    Add `className="print-week-label"` to the week-range label div in PrintGrid.
