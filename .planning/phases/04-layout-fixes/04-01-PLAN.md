<feature>
  <name>New computeSlotLayout — interval-graph coloring for slots 1–4</name>
  <files>src/lib/slotLayout.ts, src/lib/slotLayout.test.ts</files>

  <behavior>
    The new algorithm assigns every event (single-day AND multi-day) to a slot in 1–4.
    Slot 0 and MULTI_DAY_SLOT are removed entirely.

    **Key invariant for multi-day events:**
    A multi-day event gets ONE slot number, and that slot is "occupied" on every calendar date
    from its `date` (start) through its `endDate` (inclusive). When rendering, WeekRow passes
    the slotMap to DayColumn, which looks up the event by its id and renders it in the correct slot.

    **Interval-graph coloring algorithm:**
    1. Sort all events by their start date (ascending), then by duration descending (longer first
       so multi-day events get lower slot numbers preferentially — reduces fragmentation).
    2. For each event, compute its occupied date range:
       - Single-day: [date, date]
       - Multi-day: [date, endDate]
    3. Try slots 1, 2, 3, 4 in order. A slot is "available" if no already-assigned event in that
       slot has a date range that overlaps the candidate's date range.
       Two ranges [a0,a1] and [b0,b1] overlap if a0 <= b1 AND b0 <= a1 (ISO string comparison).
    4. Assign the first available slot. If slots 1–4 are all occupied → assign -1 (overflow).
    5. Return Map<eventId, slotNumber>.

    **Test cases (RED phase — write these first, all must fail against old code):**

    - Case A: Single multi-day Mon–Wed → slot 1
    - Case B: Two overlapping multi-day events (both Mon–Wed) → slots 1 and 2
    - Case C: Two non-overlapping multi-day events (Mon–Tue and Thu–Fri) → both slot 1
    - Case D: Multi-day Mon–Wed in slot 1; single-day Mon → slot 2 on Mon (slot 1 blocked by multi-day)
    - Case E: Multi-day Mon–Wed in slot 1; single-day Thu (no overlap) → slot 1 on Thu
    - Case F: 4 overlapping multi-day events all spanning the same week → slots 1,2,3,4
    - Case G: 5th overlapping event → -1 (overflow)
    - Case H: Empty input → empty Map
    - Case I: Mixed — multi-day Mon–Wed (slot 1), single-day Mon (slot 2), single-day Mon (slot 3), single-day Tue (slot 2 — no conflict with Tue)
    - Case J: Single single-day event → slot 1 (regression: old behavior unchanged for single-day)
    - Case K: 4 single-day events on same date → slots 1,2,3,4 (regression)
    - Case L: 5 single-day events on same date → 4 in slots 1-4, 1 overflow (-1) (regression)
    - Case M: weekStart/weekEnd params accepted (may be unused internally, but signature preserved)

    **Signature (unchanged from caller's perspective):**
    ```typescript

      events: LayoutEvent[],
      weekStart: Date,
      weekEnd: Date,
    ): Map<string, number>
    ```

    **Exports to keep:** `LayoutEvent`, `computeSlotLayout`, `MAX_SLOTS` (update value to 4, the max real slot)
    **Exports to remove:** `MULTI_DAY_SLOT` (removing concept entirely)
  </behavior>
