<task type="auto">
  <name>Task 1: Move sticky header inside scroll container to share column widths</name>
  <files>src/components/CalendarGrid.tsx</files>
  <action>
    The root cause of column drift: the sticky header div and the scrollable body div are sibling
    flex children. The scrollable body may reserve space for the scrollbar (browser-dependent), while
    the header does not. This makes the usable column width differ.

    **Fix strategy — ensure matching widths:**

    Move the header INSIDE the scroll container (as a sticky child of the scrolling element) rather
    than a sibling flex child above it. Both the sticky header and the WeekRows then share exactly
    the same container width.

    **Implementation:**

    In CalendarGrid.tsx, change from:
    ```tsx
    <div className="flex flex-col h-screen overflow-hidden">
      <div className="grid grid-cols-7 sticky top-0 z-10 ...">  {/* header — sibling ABOVE body */}
        ...
      </div>
      <div className="overflow-y-auto flex-1 pb-28">            {/* body */}
        {weeks.map(...WeekRow...)}
      </div>
      <RecurringFooter />
      ...
    </div>
    ```

    To:
    ```tsx
    <div className="flex flex-col h-screen overflow-hidden">
      <div className="overflow-y-auto flex-1 pb-28" ...>   {/* body — takes full flex space */}

        {/* Sticky header INSIDE the scrolling container */}
        <div
          className="grid grid-cols-7 sticky top-0 z-10 bg-white border-b border-gray-100"
          style={{ alignSelf: 'start' }}
        >
          {DAY_NAMES_SHORT.map((short, i) => (
            <div key={i} className="text-center py-2">   {/* py-2 instead of py-1 */}
              <span className="sm:hidden text-[11px] text-gray-500 font-medium">{short}</span>
              <span className="hidden sm:inline text-xs text-gray-500 font-medium">{DAY_NAMES_FULL[i]}</span>
            </div>
          ))}
        </div>

        {/* Week rows follow immediately in same container */}
        {weeks.map((week) => (
          <WeekRow ... />
        ))}
      </div>

      <RecurringFooter />
      {/* multiDayPopover EventPopover unchanged — stays outside scroll div */}
    </div>
    ```

    **Typography improvements on header cells:**
    - `py-2` (was `py-1`) — more breathing room
    - Add `font-medium` to both short and full day name spans — slightly more legible

    **Critical constraints from STATE.md — MUST preserve:**
    - `alignSelf: start` on the sticky header div (prevents CSS Grid stretch from breaking position:sticky)
    - `pb-28` on the scroll container (prevents RecurringFooter overlap)
    - `onPointerUp` handler on the scroll div (drag-to-create support) — keep it
    - `RecurringFooter` stays outside the scroll container (fixed at bottom of viewport)
    - `EventPopover` for multi-day stays outside scroll container — no change
  </action>
